# YesikNote MVP Phase 1 범위

## 목표

Owner가 예식 기본정보와 진행 방식을 입력하고 전체 식순을 편집하면 규칙 기반 사회자 대본, 진행 큐, 주의사항을 생성하고 MC 읽기 전용 화면에서 확인할 수 있게 한다.

## 포함

- 신랑·신부 기본정보와 예식장 제공 정보 임시 입력
- 주례 없음 13개, 주례 있음 15개 기본 식순
- 식순 및 화촉점화 Child Item 정렬
- 진행/미진행, 추가, 복제, 삭제, 이름 변경
- 소개 멘트, 직접 대본, 전체·식순별 요청사항
- 규칙 기반 대본과 식전 15/10/5분 체크리스트
- 실시간 미리보기와 최종 Validation
- 한국어 IME 안전 localStorage 저장·복원
- MC 읽기 전용 프롬프터

## 제외

인증, Supabase, 실제 공유 토큰, 관리자 화면, 파일·음원 업로드, 공동 편집, Snapshot, Patch, Audit Trail, PWA, 하객 기능, 청첩장, 결제, 자유 생성형 AI는 구현하지 않는다.

## 추가 확정 정책

- `religious`, `custom`은 템플릿 준비 중 상태로 표시하고 기본 식순·대본을 만들지 않는다.
- 자유 식순은 제목, 설명, MC 대본을 직접 입력한다.
- 예물교환은 기본 `active = true`지만 선택 식순이다. 비활성 시 전체 출력에서 제외하고 재활성 시 입력과 화동 설정을 복원한다.
