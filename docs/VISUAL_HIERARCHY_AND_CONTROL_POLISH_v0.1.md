# YesikNote Visual Hierarchy & Control Polish v0.1

상태: **PROPOSED**

검토일: 2026-07-24

범위: W5.1 저위험 화면 위계·조작·접근성 개선

## 1. 목적

기존 데이터 계약과 사용자 Draft를 바꾸지 않고, Owner 작성 화면부터 예식장 요약과 MC 읽기 화면까지 식순명·역할·값·상태·조작의 구분을 더 분명하게 만든다.

## 2. 자동 수정한 저위험 문제

- 전체 식순 카드의 식순명을 18~19px로 높이고 보조 설명과 상태 배지를 분리했다.
- `진행`, `확인 필요`, `미진행`을 공통 배지 스타일로 통일했다.
- 미진행 카드는 전체 투명도를 낮추지 않고 중립 배경과 선으로 구분해 식순명을 읽을 수 있게 했다.
- 미진행 상세 화면의 안내를 사용자 중심 문장으로 바꾸고 `다시 진행하기` 버튼을 추가했다. 기존 입력값은 그대로 유지된다.
- 순서 이동 버튼에 44px 터치 영역, 비활성 상태, 접근 가능한 이름과 툴팁을 적용했다.
- 복제·진행 상태 변경·삭제 버튼의 시각적 성격을 분리했다.
- 모바일에서 다섯 조작 버튼이 한 줄에 겹치지 않도록 2열 또는 3열로 줄바꿈했다.
- 예식장 체크표에서 혼인서약·성혼선언·덕담/축사의 역할 라벨과 사람 이름을 분리했다.
- 축가·축무는 곡 수, 진행자, 곡명을 별도 줄로 표시한다.
- 키보드 `focus-visible` 표시를 버튼·링크·입력·선택·요약 요소에 공통 적용했다.
- 390px 헤더에서 `예식노트` 브랜드명이 부자연스럽게 줄바꿈되지 않도록 했다.

## 3. 검증한 화면

- Owner 전체 식순과 상세 설정
- Owner 최종 확인
- 예식장 전달용 체크표 미리보기
- 최종 예식 식순표
- MC 읽기 화면의 대본·Cue·Note
- 390×844, 768×1024, 1280×900, 1440×900

모든 확인 해상도에서 문서 너비의 수평 넘침은 발견되지 않았다. 전체 식순 조작 버튼의 최소 높이는 44px, MC 하단 조작 버튼은 58px였다. MC 대본 본문은 화면 너비에 따라 22~28px였고 Cue와 Note는 서로 다른 배경색과 제목으로 구분됐다.

## 6. Product Owner 결정 필요

1. 모바일 조작을 현재의 명시적 2열 버튼으로 유지할지, 후속 Batch에서 `더보기` 메뉴로 줄일지 결정이 필요하다.
2. 공개 GitHub Pages에서 `/mc`를 직접 새로 열면 404가 발생한다. SPA fallback 또는 배포 경로 처리 변경은 배포 설정·Router 범위이므로 별도 승인이 필요하다.
3. Pretendard를 모든 기기에서 동일하게 보장하려면 폰트 자산 또는 외부 제공 방식에 대한 결정이 필요하다.
4. 실제 프린터·브라우저 인쇄 미리보기에서 A4 페이지 나눔을 확인하는 대표 QA가 필요하다. 이번 검증은 기존 A4 print CSS, build, 화면 구조까지다.
