# Acceptance Criteria

## 템플릿과 편집

- [ ] 주례 없음은 정확히 13개, 주례 있음은 정확히 15개다.
- [ ] 주례 소개 기본 위치는 화촉점화 다음이다.
- [ ] 모든 최상위 식순과 화촉 Child Item을 이동할 수 있다.
- [ ] 모바일 390px에서 위/아래 버튼으로 이동할 수 있다.
- [ ] 혼인서약·예물교환·성혼선언은 자유롭게 재정렬되며 비권장 순서는 warning만 표시한다.
- [ ] 비활성은 삭제와 구분되고 재활성 시 입력값이 복원된다.

## 대본 분기

- [ ] 식전 15/10/5분 안내와 휴대전화 무음/진동 문구가 생성된다.
- [ ] 식전 방송에는 피로연 장소가 없고 폐회식에만 표시된다.
- [ ] 화촉점화 어머님·부모님 A·부모님 B Child 구조가 정확하다.
- [ ] 신랑 입장 2개와 신부 입장 4개 표준 조합이 분기된다.
- [ ] 주례 유무에 따라 맞절·혼인서약·성혼선언 진행자가 분기된다.
- [ ] 화동, 덕담·축사, 공연, 혼주 참석·호칭·포옹 분기가 적용된다.
- [ ] 한글 받침에 맞는 조사가 출력된다.

## 추가 정책

- [ ] `religious`, `custom`은 템플릿 준비 중이며 기본 대본을 생성하지 않는다.
- [ ] 활성 자유 식순은 제목, 설명, MC 대본이 모두 필요하다.
- [ ] 예물교환은 기본 진행이고 미진행은 Validation 오류가 아니다.
- [ ] 예물교환 미진행 시 MC·최종 대본·큐·화동이 제외된다.

## 저장과 MC

- [ ] 한국어 IME 조합 중 저장하지 않고 종료 1초 후 저장한다.
- [ ] 새로고침 후 단계, 선택, 순서, Child, 직접 대본이 복원된다.
- [ ] 전체 요청사항과 식순별 실행 메모가 구분된다.
- [ ] MC 화면에는 편집 기능이 없고 Owner의 실제 순서를 따른다.
- [ ] MC 이전·완료·다음, 다크모드, 글자 크기, 스크롤 잠금이 동작한다.

## 품질

- [ ] `npm run test` 결과를 보고한다.
- [ ] `npm run build`가 성공한다.
- [ ] API 키, 토큰, 개인정보, `.env`가 커밋되지 않는다.
