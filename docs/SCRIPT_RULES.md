문서 목적:
- 신랑·신부가 체크와 짧은 입력만으로 식순을 만들 때 사용되는 표준 대본 원본
- Codex의 `scriptEngine.ts`가 사용하는 조건, 변수, 진행 큐, 주의사항 기준
- 문장 맞춤법과 띄어쓰기는 교정하되, 이동주 Product Owner가 작성한 현장 말투와 의도를 유지한다.

## 1. 핵심 제품 정책

### 1.1 작성 절차와 실제 예식 식순은 분리한다

Owner 작성 절차:
1. 예식 기본정보
2. 예식 유형 및 기본 식순 선택
3. 전체 식순 편집
4. 항목별 상세 설정 및 대본 확인
5. 최종 확인 및 사회자 공유

실제 예식 식순은 아래 기본값을 불러오되 모든 항목을 이동·추가·삭제·이름 변경할 수 있다.

### 1.2 주례 없는 예식 기본 식순

1. 개식사
2. 화촉점화
3. 신랑 입장
4. 신부 입장
5. 신랑·신부 맞절
6. 혼인서약
7. 예물교환
8. 성혼선언
9. 덕담/축사
10. 축가/축무/축주
11. 양가 부모님 및 내빈께 인사
12. 신랑·신부 행진
13. 폐회식

### 1.3 주례 있는 예식 기본 식순

1. 개식사
2. 화촉점화
3. 주례 소개(등단)
4. 신랑 입장
5. 신부 입장
6. 신랑·신부 맞절
7. 혼인서약
8. 예물교환
9. 성혼선언
10. 주례사
11. 덕담/축사
12. 축가/축무/축주
13. 양가 부모님 및 내빈께 인사
14. 신랑·신부 행진
15. 폐회식

정책:
- 주례 소개는 기본적으로 화촉점화 다음, 신랑·신부 입장 전에 둔다.
- 예식장 운영 방식에 따라 주례 소개를 신랑·신부 입장 뒤로 이동할 수 있다.
- 혼인서약 → 예물교환 → 성혼선언은 권장 기본값이며 사용자가 위아래로 이동할 수 있다.
- 일반적이지 않은 순서는 경고만 표시하고 저장을 차단하지 않는다.

### 1.4 모든 식순에서 제공할 입력

각 식순 카드에는 다음 입력을 제공한다.

- 사용 여부
- 진행자 또는 참여자
- 기본 멘트 사용
- 소개 멘트 추가
- 전체 대본 직접 수정
- 사회자 요청사항
- 진행 큐
- 예상 소요시간
- 위로 이동 / 아래로 이동 / Drag & Drop
- 복제 / 삭제 / 이름 변경

화면 용어 변환:
- Owner 화면: `사회자 요청사항`
- MC 화면: `주의사항 / 실행 메모`
- `소개 멘트 추가`는 사회자가 실제로 읽는 문장이다.
- `사회자 요청사항`은 읽지 않고 현장에서 확인하는 메모다.

### 1.5 요청사항 입력 정책

두 종류를 모두 제공한다.

1. 전체 요청사항
   - 예식 전체에 적용되는 내용
   - MC 화면 상단에 고정 표시
   - 예: 특정 호칭 금지, 신부 부모님 건강 상태, 예식 전체 시간 단축

2. 식순별 요청사항
   - 해당 식순에서만 필요한 내용
   - 관련 대본 카드의 주의사항으로 표시
   - 예: 신부 아버님 보행 속도, 혼주님은 앉은 상태로 인사, 음원 시작 시점

### 1.6 소개 멘트 정책

주례, 성혼선언자, 덕담자, 축사자, 축가자, 축무자, 축주자, 화동처럼 소개가 필요한 식순은 다음 중 하나를 선택한다.

- 기본 소개 멘트 사용
- 직접 소개 멘트 입력

활성화된 소개 대상은 이름 또는 호칭이 필수다.
직접 소개를 선택했는데 입력하지 않으면 최종 공유를 막고 `소개 멘트를 입력해 주세요`를 표시한다.
양가 부모님 입장에는 선택형 `특별 소개 멘트`를 추가할 수 있다.

## 2. 공통 변수

- `{{wedding_date}}`: 예식일 전체 표기
- `{{wedding_year}}`, `{{wedding_month}}`, `{{wedding_day}}`
- `{{groom_name}}`: 신랑 이름
- `{{bride_name}}`: 신부 이름
- `{{officiant_name}}`: 주례 선생님 성함
- `{{officiant_bio}}`: 주례 약력 또는 소개
- `{{speaker_name}}`: 성혼선언·덕담·축사 진행자
- `{{speaker_relation}}`: 진행자와 신랑·신부의 관계
- `{{performer_name}}`: 축가·축무·축주 진행자
- `{{performance_title}}`: 곡명 또는 공연명
- `{{flower_child_name}}`: 화동 이름 또는 표시 호칭
- `{{flower_child_intro}}`: 화동 소개 멘트
- `{{banquet_location}}`: 피로연 장소
- `{{photo_guide}}`: 사진 촬영 안내
- `{{custom_intro}}`: 식순별 추가 소개 멘트
- `{{global_request_note}}`: 전체 요청사항
- `{{item_request_note}}`: 식순별 요청사항

호칭:
- 한쪽 부모만 참석하거나 성별·관계가 확실하지 않으면 `혼주님`을 사용한다.
- 신랑·신부가 입력한 사용자 지정 호칭이 있으면 해당 호칭을 우선한다.

## 3. 출력 모델

모든 실행 대본은 아래 구조를 사용한다.

```ts
type ScriptSection = {
  id: string;
  parentId?: string;
  title: string;
  narration: string;
  cue: string[];
  note: string[];
  orderPath: number[];
  estimatedTimeSeconds?: number;
};
```

우선순위:
1. 사용자가 전체 대본을 직접 수정한 `narrationOverride`
2. 기본 대본 + 입력한 소개 멘트
3. 예식노트 표준 기본 대본

# 4. 식전 안내방송

## 적용 시점

- 예식 15분 전
- 예식 10분 전
- 예식 5분 전
- 총 3회 자동 체크리스트로 표시한다.
- v0.1 프로토타입에서는 실제 시간 알림을 보내지 않고 체크리스트와 대본만 제공한다.

## 사회자 낭독문

잠시 로비에 계신 하객 여러분께 안내 말씀드립니다.  
잠시 후 신랑 {{groom_name}} 군과 신부 {{bride_name}} 양의 결혼식이 진행될 예정입니다.  
아직 로비에 계신 하객 여러분께서는 지금 바로 식장 안으로 입장하시어 앞쪽 빈자리부터 착석해 주시면 대단히 감사하겠습니다.  
원활한 예식 진행을 위해 휴대전화는 무음 또는 진동으로 전환해 주시길 부탁드립니다.

## 진행 큐

- 15분 전 1차 안내
- 10분 전 2차 안내
- 5분 전 3차 안내
- 하객 착석 상태 확인
- 홀 문, 음향, 조명, 입장 대기 상태 확인

## 주의사항

- 피로연 및 식사 장소는 식전 안내에서 말하지 않는다.
- 피로연 안내는 폐회식에서만 제공한다.
- 5분 전 방송 후 예도팀 또는 예약실에 개식 가능 여부를 확인한다.

# 5. 개식사

## 공통 도입

내빈 여러분, 안녕하십니까.  
오늘 {{wedding_year}}년 {{wedding_month}}월 {{wedding_day}}일, 신랑 {{groom_name}} 군과 신부 {{bride_name}} 양의 결혼식에 참석해 주신 내빈 여러분께 감사의 말씀을 드립니다.

## 주례 없는 예식일 때만 삽입

오늘 예식은 특별하게 주례 없는 예식으로 진행될 예정입니다.

## 공통 마무리

축복된 예식을 위해 진행 중에는 조용한 분위기와 경건한 마음으로 두 사람의 소중한 시작을 함께 축복해 주시길 부탁드립니다.  
그럼 지금부터 여러분의 큰 박수와 함께 오늘의 예식을 시작해 보도록 하겠습니다.  
내빈 여러분, 큰 박수 부탁드립니다.

## 조건

- `ceremonyType === 'no_officiant'`: 주례 없는 예식 문장 삽입
- `ceremonyType === 'officiant'`: 해당 문장 생략

## 진행 큐

- 모든 주요 참여자 준비 확인
- 홀 문 닫힘 확인
- 음향·조명 준비 확인
- 개식 선언 후 다음 실제 식순 준비

## 주의사항

- 신랑·신부 이름과 날짜를 최종 확인한다.
- 개식사가 화촉점화 뒤로 이동했을 경우 다음 식순 큐를 실제 정렬 순서 기준으로 표시한다.

- 이동이 불편한 혼주가 있으면 식순별 요청사항을 MC 화면에 강조한다.

# 16. 축가/축무/축주

## 데이터 구조

공연은 1개 이상 배열로 관리한다.

```ts
type PerformanceItem = {
  id: string;
  type: 'song' | 'dance' | 'instrumental';
  performerName: string;
  performerRelation?: string;
  title?: string;
  introText?: string;
  samePerformerAsPrevious: boolean;
  order: number;
  requestNote?: string;
};
```

화면 표시 예:
- 축가 1곡
- 축가 2곡
- 축가 1곡 + 축무 1곡
- 축주 1곡

## 첫 번째 공연

다음은 두 사람을 위한 {{performance_type_ko}}가 준비되었습니다.  
오늘 {{performance_type_ko}}는 {{performer_name}}께서 준비해 주셨습니다.  
{{custom_intro}}  
큰 박수로 맞이해 주시길 바라겠습니다.

## 두 번째 이후, 다른 진행자

이어서 {{performer_name}}께서도 두 사람을 위해 {{performance_type_ko}}를 준비해 주셨습니다.  
{{custom_intro}}  
큰 박수로 맞이해 주시길 바라겠습니다.

## 두 번째 이후, 같은 진행자

이어서 같은 분께서 두 사람을 위해 두 번째 {{performance_type_ko}}를 준비해 주셨습니다.  
계속해서 큰 박수 부탁드립니다.

## 용어 변환

- `song` → 축가
- `dance` → 축무
- `instrumental` → 축주

## 진행 큐

- 진행자 및 인원 대기 확인
- 마이크·악기·무대 배치 확인
- 음원 또는 MR 준비 확인
- 공연 유형과 순번 표시
- 공연 종료 후 박수 유도
- 같은 진행자의 다음 곡 여부 확인

## 주의사항

- 소개 대상 이름 또는 호칭은 필수다.
- `기본 소개 사용` 또는 `직접 소개 입력`을 선택하게 한다.
- 곡명, 공연명, MR 정보는 v0.1에서 텍스트로만 입력한다.
- 파일 업로드와 재생 기능은 이번 범위에서 제외한다.

# 17. 양가 부모님 및 내빈께 인사

신부 부모님께서는 자리에서 일어나 신랑 신부를 한 번 안아 주시길 바라겠습니다.

이제 이어서 신랑 부모님께도 인사드리겠습니다.

신랑 부모님 앞에 정렬되면:

서로 사랑하는 마음 변치 않고 예쁘게 알콩달콩 잘살도록 하겠습니다.  
신랑 신부, 신랑 부모님께 인사!

신랑 부모님께서는 자리에서 일어나 신랑 신부를 한 번 안아 주시길 바라겠습니다.

## 내빈께 인사

이어서 내빈 여러분께 인사드리겠습니다.  
두 사람은 중앙으로 자리를 이동해 주시길 바라겠습니다.

중앙 정렬 후:

저희 신랑 신부를 위해 먼 길 찾아주시고 축복해 주셔서 진심으로 감사드립니다.  
식을 마치고 난 후 한 분 한 분 직접 찾아뵙고 인사드리는 것이 도리이나, 먼저 이 자리에서 감사의 인사를 드리겠습니다.  
신랑 신부, 내빈께 인사!

## 혼주 참석·호칭 설정

각 가정별로 아래를 선택한다.

- 부모님 두 분
- 아버님 한 분
- 어머님 한 분
- 혼주님 한 분
- 참석하지 않음
- 사용자 지정 호칭

대본은 참석 인원과 사용자 지정 호칭에 맞게 바뀐다.

## 진행 큐

- 신부 측 혼주 위치와 참석 상태 확인
- 신랑·신부 신부 측 이동
- 정렬 완료 후 인사 구령
- 포옹 가능 여부 확인
- 신랑 측 이동
- 정렬 완료 후 인사 구령
- 포옹 가능 여부 확인
- 중앙 이동
- 내빈 인사

## 주의사항

- 몸이 불편하거나 자리에서 일어나기 어려운 혼주는 포옹 안내를 자동 생략할 수 있다.
- 신랑·신부의 이동 속도와 혼주 건강 상태를 식순별 요청사항으로 표시한다.

- 참석하지 않는 쪽에 대한 인사 식순은 비활성화할 수 있다.

# 20. 저장 및 화면 정책

## 저장

- 한국어 IME composition 중 저장하지 않는다.
- composition 종료 후 1초 debounce로 localStorage에 저장한다.
- `저장 중 / 저장됨 / 저장 실패`와 마지막 저장 시각을 표시한다.
- 실패해도 입력 내용을 삭제하지 않는다.
- 전체 식순과 중첩 child items 순서를 복원한다.

## MC 화면

- 읽기 전용
- 현재 순서와 다음 순서 Sticky
- 큰 낭독문
- 파란색 계열 진행 큐
- 주황색 계열 주의사항
- 전체 요청사항 상단 고정
- 식순별 요청사항은 해당 카드에 표시
- 이전 / 완료 / 다음 큰 버튼
- 다크모드
- 글자 크기
- 스크롤 잠금
- Owner의 실제 정렬 순서와 완전히 동일하게 표시
